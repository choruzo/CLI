Las tres piezas

decisions.jsonFuente de verdad (DecisionStore)
vectors.dbÍndice de búsqueda (VectorStore: sqlite-vec o brute-force JS)
EmbeddingsHTTP /v1/embeddings → fallback ONNX local
Invariante: decisions.json nunca se pierde. Si el embedder o el índice vectorial fallan, la decisión igual se persiste; solo se degrada la búsqueda semántica hasta el próximo reindex(). La memoria es auxiliar: jamás tumba una sesión.
